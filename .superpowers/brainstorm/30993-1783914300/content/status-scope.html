A — Chỉ làm status trực quan hơn

Giữ nguyên trạng thái và hành vi; cải thiện tên gọi, màu, icon, mô tả và hierarchy.

Open→ Agent running→ Human review→ Merging→ Done

Mỗi status nói rõ: ai đang giữ việc, điều gì đang chờ, hành động tiếp theo là gì.

B — State machine kiểu Symphony

Thiết kế lại status/bucket để board trở thành control plane trực quan, nhưng tận dụng runner và dữ liệu hiện có.

C — Áp dụng đầy đủ mô hình OpenAI

Ngoài UI còn đổi orchestration: tự pick task, chạy đồng thời, retry/reconcile và quản lý dependency DAG.

Khuyến nghị ban đầu: B. Nó đúng với điều bạn thích ở bài viết — status trực quan và task-centric — đồng thời không kéo scope sang một orchestrator mới khi repo đã có Symphony runner.
